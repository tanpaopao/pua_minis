# Changelog

All notable changes to `pua-minis` will be documented in this file.

## [v1.1.0] - 2026-03-21

### Added
- Added **path-switching** reference to define what counts as a truly different troubleshooting path.
- Added **failure-exit** reference for structured, evidence-based failure handoff.
- Added **status-format** reference for lightweight state blocks in complex troubleshooting.
- Added **self-check** reference for mid-task anti-spinning self-review.
- Added **reverse hypothesis** as an explicit troubleshooting action when the main hypothesis fails twice.
- Added lightweight status block and mid-task self-check guidance to the main skill.
- Added structured failure-exit protocol to the main skill and examples.

### Changed
- Upgraded `SKILL.md` from v1.0.0 to **v1.1.0**.
- Strengthened escalation guidance across **L1 / L2 / L3**.
- Clarified that repeated parameter tweaking does **not** count as path switching.
- Expanded examples to cover reverse-hypothesis usage and qualified failure exit.
- Expanded checklist to cover path switching, reverse hypothesis, failure-exit preconditions, and lightweight status blocks.
- Updated README to reflect v1.1.0 capabilities and full reference set.

### Kept
- Preserved the Minis-first style: **less talk, more action, stronger closure**.
- Preserved the skill focus on debugging, troubleshooting, deployment, config, API, ops, and data-failure scenarios.

## [v1.0.0]

### Initial release
- Initial Minis-native high-agency troubleshooting skill.
- Included baseline trigger rules, escalation ladder, checklist, and examples.
