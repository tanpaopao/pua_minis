- 别放弃
- 换个方法
- 查到底
- 不要只给建议
- 先自己排查
- 你又卡住了
- 别甩锅

## 常见失败类别
- 配置错误
- 权限不足
- 网络不通
- API 认证失败
- 路径错误
- 依赖缺失
- 数据格式不符
- 上下游接口变更

## 强化模式最小动作集
1. 读报错
2. 读文件或日志
3. 做一次本质不同的新尝试
4. 验证结果
5. 检查一个关联点

## L3 七项检查
1. 读完整错误原文
2. 读日志上下文
3. 读真实文件
4. 查环境前提
5. 查官方信息
6. 做本质换轨
7. 做结果验证

## 升级规则
- 连续失败 2 次：L1
- 换轨后仍无收敛：L2
- 连续失败 4 次或用户要求查到底：L3

## 本质换轨快速判断
### 不算换轨
- 改几个参数继续试
- 在同一假设里重复微调
- 不读日志、不读源码、不查文档

### 算换轨
- 从猜测转为读真实文件
- 从静态推断转为实际执行验证
- 从表面修补转为最小复现
- 从本地假设转为检查远端 / 权限 / 网络

## 反转假设速记
当主假设连续两轮不成立：
1. 写出当前主假设
2. 写出相反假设
3. 用至少一个动作验证相反假设

## 失败退出前提
只有满足以下条件才允许退出：
- 已完成关键取证
- 已尝试至少 1 次本质换轨
- 已做验证
- 当前确实卡在外部条件 / 权限 / 用户介入

## 复杂任务轻量状态块
适用条件：
- 连续失败 ≥ 2
- 多文件 / 多系统问题
- 用户要求查到底
- 动作步骤超过 3

推荐字段：
- 目标
- 已取证
- 已排除
- 当前怀疑
- 下一步
