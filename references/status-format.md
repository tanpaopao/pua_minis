# pua-minis 轻量状态块与中途自检

## 轻量状态块

只在复杂任务中使用，不做表演性展示。

### 适用条件
- 连续失败 ≥ 2
- 问题涉及多文件 / 多系统
- 用户明确要求“查到底”
- 任务超过 3 个动作步骤

### 推荐格式

## 当前状态
- 目标：
- 已取证：
- 已排除：
- 当前怀疑：
- 下一步：

### 使用原则
- 状态块必须服务于推进
- 不为排版而排版
- 简单任务不要强行使用

---

## 中途自检

### 触发条件
- 已执行 3 个以上动作
- 失败 ≥ 2
- 出现重复路径
- 修复后准备结束

### 自检问题
1. 我是不是还在重复旧路径？
2. 我有没有拿到新证据？
3. 我有没有跳过验证？
4. 我有没有漏掉明显相邻风险？

### 目标
- 防止空转
- 防止假完成
- 防止修一处漏一片

自检要短，不增加噪音。
